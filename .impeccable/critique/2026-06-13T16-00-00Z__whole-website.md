## Anti-Patterns Verdict

Terminal-luxury aesthetic continues to clear the slop test on both passes. Dark + mono + lime for developer tooling is becoming saturated in 2026; warm hue-60 tint and workbench density are the differentiators that hold. No absolute bans triggered.

Detector findings:
- `tiny-text` at `--fs-xxs` (10px) — real. Segment labels and props grid captions fall below the 11px practical minimum. Affects `ValidatorSegCard` and `ValidatorPropsGrid`.
- `line-length ~86ch` — real. UUID list on generator at full-width viewport. Already capped in prose but not in the UUID output grid.
- `gradient-text` (2×) — **false positive**. Source is Tailwind's `@layer properties` block (`--tw-gradient-from`, `--tw-gradient-via`, `--tw-gradient-to` declared as CSS custom properties in the bundle). No `background-clip: text` in application CSS.
- `bounce-easing` (1×) — **false positive**. Tailwind `@layer properties` block also declares animation-related custom properties; none map to bounce easing in `tokens.css`. Confirmed: `--ease-standard` is `cubic-bezier(0.2, 0.7, 0.3, 1)`, no bounce.
- `overused-font` — false positive in context (Geist Mono is a technical requirement for UUID legibility and zero-disambiguation).

## Priority Issues

[P0] Generator format toggles use `<label>+hidden checkbox` pattern while validator option toggles use `<button aria-pressed>`. These are semantically different controls doing the same job. Standardise both to `<button aria-pressed>` (the validator's existing pattern) so keyboard and screen-reader behaviour is identical across both tools.

[P1] Empty validator right pane. When input is cleared the right column goes blank — no empty-state message, no hint. Users get no signal that the tool is ready and waiting. Add a muted placeholder: e.g. "paste a UUID to inspect" at centre of the empty pane.

[P1] `100% confidence` widget in ValidationBanner is semantically wrong. UUID validation is binary (format pass/fail), not probabilistic; no ML score exists. The widget implies confidence scoring that doesn't happen. Remove it; the "valid · UUID vN" banner line already communicates success unambiguously.

[P1] Default batch size is 1 — the generator panel opens to a single UUID. This undersells the batch feature that is the tool's primary differentiator. Set default to 8.

[P1] `tiny-text`: `--fs-xxs` (10px) appears in 7 CSS selectors including segment captions and props grid hints. 10px is below the practical accessibility minimum. Raise to `--fs-xs` (11px) or `--fs-sm` (12px).

[P2] Keyboard shortcut hints hardcoded as Mac glyphs (`⌥`, `⌘`, `⇧`). On Windows / Linux these are meaningless. Detect platform and swap to `Alt`, `Ctrl`, `Shift` equivalents, or use platform-neutral descriptions.

[P2] ValidatorPropsGrid: 5 of 9 rows describe the *input format* (format, length, lowercase, hyphens, braces), not the UUID itself. These are low-signal for most users who already know how they typed the value. Consider hiding format-descriptor rows by default, or collapsing into a single "input" row.

[P3] Page `<title>` is `uuid-generator` — matches the repo name, not the product. Update to `uuidlab — UUID Generator & Validator` to improve discoverability and bookmark clarity.

[P3] Validator input has a permanent accent-coloured border even when empty. This false-signals an active or focused state on first load. Use `--seam` for the idle border; switch to `--accent` only on focus or when input is non-empty.

## Persona Red Flags

**Alex (Power User):** Batch default of 1 requires an extra interaction every session. No pin-UUID-across-regeneration means bulk workflows must regenerate everything or manually rescue values.

**Sam (Accessibility):** Generator format toggles are checkboxes (`role=checkbox`) while validator options are toggle buttons (`role=button` + `aria-pressed`). Screen reader announcements differ. `--fs-xxs` (10px) labels likely fail AT zoom tolerances on mobile.

**Jordan (First-Timer):** `uppercase()`, `stripHyphens()`, `wrapBraces()` use function-call syntax that signals developer familiarity. No inline preview of what each format option produces. Empty validator pane gives no entry cue.

## Minor Observations

- `ValidationBanner` uses `role="status"` but there is no ARIA live region on the validation pane wrapper; updates may not be announced by screen readers on older AT.
- Generator shortcut `⌥↑/↓` for batch increment: the ↑/↓ glyphs render at different baseline heights across OS; consider `⌥+` / `⌥−` as fallback.
- `text-wrap: balance` not applied to `h1` inside the hero — fixed in prose but not headings.
